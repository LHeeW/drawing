```markdown
# 프로젝트명

- drawing

- npm install
- npm run dev

- React, TypeScript, Vite, Fetch
- Tanstack Query, Zustand, React Router Dom, React Zoom Pan Pinch

## 구현 기능

-[x] 전체 배치도에 평면도의 위치를 표시하고 클릭하면 해당 도면으로 이동
-[x] 해당 도면의 이름과 그 도면과 관련된 공종을 navigation으로 구현
-[x] 해당 도면과 관련된 공종을 선택하여 공종의 도면을 표시 (공종 다중선택 가능)
-[x] 공종 선택시 사이드 바가 나타나며 리비전이 존재하는 공종은 이전 버전과 변경 사항을 확인 할 수 있도록 구현
-[x] 화면을 줌 인/아웃 하며 도면을 확대/축소 해서 볼 수 있고, 화면을 이동시켜 잘리는 부분 없이 도면의 모든 부분을 볼 수 있게 구현
```

### C. DESIGN.md

- **데이터 분석**: 전체 배치도를 기준으로 건물 혹은 위치별 평면도를 가지고 또 그 내부에서 공종별로 나뉘는 트리 구조로 형성되었다고 파악했습니다. 또한 현 데이터는 연관된 데이터를 모두 연결지어 가공되지 않은채 그대로 데이터로 뿌려주는 구조로 만약 공종별로 리비전이 수천개가 된다거나 transform이 굉장히 많이 진다면 클라이언트에서 모든 로직을 가공하는데 있어 과부하가 올 것으로 예상되어 일부 계산영역(transform을 서버에서 절대좌표로 미리 계산해준다거나 하는 등)을 서버에서 미리 가공해주어 적절히 서버와 클라이언트의 로직을 분리해주는게 좋을거 같다는 생각이 듭니다.

- **접근 방식**: 비지니스 로직과 라우터 설정을 우선적으로 설계하여 큰 틀에서 전체적인 데이터 흐름을 구성 해놓고 UI 작업을 하였습니다. UI 작업시에는 기준이 되는 도면을 먼저 화면에 띄워주고 그것을 중심으로 UI를 최대한 직관적으로 볼 수 있도록 구성하였습니다.

- **UI 설계 결정**:

  - 처음 접하는 사람이라도 한눈에 파악 할 수 있는 직관성과 최소한의 클릭으로 많은 정보를 표현하는 간편성에 중점을 두었습니다.
    우선 첫 화면에는 전체 배치도가 표시되며 해당 도면에는 클릭이 가능한 녹색으로 표시된 공간과 이름이 띄워집니다.
    클릭하여 들어가면 선택된 도면이 띄워지고 상단에 navigation bar를 통해 관련된 공종을 선택하여 해당 공종의 도면을 따로 띄워주어 선택된 도면과 선택된 공종의 도면을 비교하면서 볼 수 있도록 설계하였습니다. 또한 공종 선택시 사이드 바가 생성되어 과거의 버전이 존재 하는지 여부도 알 수 있으며 과거 버전이 있다면 어떤 변경사항이 존재하는지도 한눈에 볼 수 있도록 설계하였습니다.

  - 처음에는 드롭다운 메뉴를 통해 도면을 선택하고 보여주는 방식으로 설계하려고 했습니다. 명시적인 도면의 이름을 선택 할 수 있다는 장점이 존재하지만 텍스트로만 이루어진 UI는 한눈에 파악하기 힘들다는 단점이 존재하였고 이 방식의 장점인 명시적인 표현 또한 도면에 직접 이름을 띄워주는 것으로 표현 가능하여 현 방식을 선택하게 되었습니다.

- **기술 선택**: 상태 관리는 현 프로젝트의 metadata를 넘어서 실무에서 방대한 양의 데이터를 받는다는 가정하에 서버/클라이언트 상태로 나누어 관리 해주었습니다.
  서버에서 받는 데이터는 React Query를 사용하여 데이터 캐싱이나 로딩,에러 처리를 원활하게 해주었습니다.
  공종을 선택하여 도면을 보여주는 로직 같이 클라이언트에서 이루어지는 것은 Zustand를 통해 props drilling을 최소화 해주었습니다.
  스타일링의 경우 컴포넌트의 구성 자체가 꽤 복잡하기에 최대한 비지니스, 컴포넌트, 스타일링을 분리해주어 관리하는게 코드를 볼때 가독성이 좋을 것 같아 CSS-Module을 선택하였습니다.

- **어려웠던 점 및 개선 방안**: revision 외에도 region이 따로 존재했던 공종이 있었는데 이 도면을 처리해주는데 굉장히 까다로워 region을 한번에 불러와서 사이드 바에서 같이 동작 하도록 구현하였습니다. 시간이 더 주어졌다면 이 region 부분을 독립적으로 동작하게 개선해보고 싶고, 또 다른 한가지는 현재 짜여진 UI는 최신 revision을 보여주고 사이드바를 통해 과거의 revision을 보여주긴하지만 최신과 과거 버전을 여러개 같이 띄울 수는 없는 구조입니다. 이것 또한 과거 버전과 현재 버전을 모두 한번에 볼 수 있는 방식도 구성해보고 싶습니다.
